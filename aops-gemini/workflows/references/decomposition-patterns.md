# Decomposition Patterns and Strategies

Common patterns for breaking down complex goals into manageable tasks.

## Temporal Patterns

- **Sequencing**: Task A must finish before Task B begins.
- **Parallelism**: Multiple tasks can be performed independently.
- **Milestones**: Grouping tasks around key delivery points.

## Functional Patterns

- **Layered**: Breaking down by technical layers (e.g., UI, backend, database).
- **Feature-Based**: Grouping tasks by user-facing features.
- **Dependency-Driven**: Identifying core bottlenecks and addressing them first.

## Complexity-Based Patterns

- **Spike**: Researching an unknown to reduce uncertainty.
- **Prototype**: Building a minimal version to validate an idea.
- **Productionization**: Moving from a prototype to a stable, tested system.

## Task Granularity Heuristics

- Aim for tasks that take 1-4 hours to complete.
- Each task should have a clear "why" and a single primary deliverable.
- Avoid over-decomposing into trivial subtasks that add more overhead than value.
